Analytics: Earnings Sensitivity (Code Sample)


// Simple NII sensitivity calculation (illustrative)

function netInterestIncome(assets, yieldRate, liabilities, costRate) {
  const interestIncome  = assets      * yieldRate;
  const interestExpense = liabilities * costRate;
  return interestIncome - interestExpense;
}

// 100 bps parallel shift scenario
const baseNII = netInterestIncome(120e9, 0.045, 100e9, 0.018);
const shockedNII = netInterestIncome(120e9, 0.055, 100e9, 0.028);

console.log("Base NII (in €):", baseNII.toLocaleString());
console.log("Shocked NII (in €):", shockedNII.toLocaleString());
This is a simplified JavaScript example showing how we might compute net interest income under a base scenario vs. a parallel rate shock. In production, this logic lives in our risk engine, not in a browser, but it's a good illustration of how we can demonstrate analytics directly within the presentation.

Key Financial Formulas

**Net Present Value (NPV):**

$$\text{NPV} = \sum_{t=1}^{T} \frac{CF_t}{(1 + r)^t} - I_0$$

**Return on Equity (ROE):**

$$\text{ROE} = \frac{\text{Net Income}}{\text{Average Shareholders' Equity}}$$

**Cost of Equity (CAPM):**

$$k_e = r_f + \beta \left( r_m - r_f \right)$$

Walk through each formula at a high level: - NPV for project and product-level investment appraisal. - ROE as a top-level profitability metric from a shareholder perspective. - CAPM-based cost of equity used in our hurdle rate and valuation models.

Risk-Adjusted Performance

  • We assess portfolios on a **risk-adjusted** basis, not just raw returns.
  • Example: **Sharpe Ratio** $$\text{Sharpe} = \frac{R_p - R_f}{\sigma_p}$$
  • Higher Sharpe Ratio ⇒ better compensation for each unit of risk.
Emphasize that headline returns are not sufficient. Risk-adjusted measures like the Sharpe Ratio provide a normalized view, especially useful when comparing strategies or desks.

Valuation: DCF Snippet (Python)


from math import pow

def npv(cash_flows, discount_rate):
    """
    cash_flows: list of cash flows [CF1, CF2, ..., CFT]
    discount_rate: annual discount rate, e.g. 0.1 for 10%
    """
    return sum(cf / pow(1 + discount_rate, t + 1)
               for t, cf in enumerate(cash_flows))

cash_flows = [25_000_000, 27_000_000, 29_000_000, 31_000_000]
discount_rate = 0.09

project_npv = npv(cash_flows, discount_rate)
print(f"Project NPV: €{project_npv:,.2f}")
